R2. Les courbes de niveau

Définition

Les courbes de niveau permettent de quantifier la profondeur de la surface. Elles sont générées en suivant trois pas d’espacement, qui peuvent varier en fonction de l’échelle de représentation (1/5, 1/10 ou 1/20 pour les blocs ornés) et la granularité de la surface. Ainsi, les valeurs choisies sont généralement soit 1, 5 et 10 mm, soit 2, 10 et 20 mm pour les surfaces ornées.

Mise en oeuvre

Note : si le modèle 3D a subi une transformation (translation ou rotation différent de "0" ; mise à l'échelle différente de 1), alors il faut appliquer toutes les transformations. Sinon les zébrures ne seront pas orientées correctement. Pour appliquer les transformations : `ctrl`+`A`.

  1. Onglet 1 : Masquer la sphère témoin.
  2. Onglet 2 : Sélectionner le maillage, appliquer le matériau `R2_CDN`.
  3. Onglet 2 : Modifier l'espacement entre les courbes de niveau.
  4. Premier passage :
    1. Onglet 2 : Sélectionner le noeud `Valeur initiale x1`.
    2. Onglet 3 : Connecter `Ouput 1` avec `R2_CDN`. Casser toute autre liaison que celle-ci.
    3. Onglet 3 : Renommer les sorties du noeud terminal en fonction de la valeur choisie à l'étape 3.
    4. Onglet 3 : Vérifier que le noeud terminal soit connecter sur la première puce
    5. Onglet 3 : Sélectionner le moteur de rendu EEVEE.
    6. Onglet 3 : Render all cameras.

  1. Deuxième passage :
    1. Onglet 2 : Sélectionner le noeud `Valeur initiale x5`.
    2. Onglet 3 : Connecter la deuxième puce du noeud terminal.
    3. Onglet 3 : Render all cameras.

  1. Troisième passage :
    1. Onglet 2 : Sélectionner le noeud `Valeur initiale x10`.
    2. Onglet 3 : Connecter la troisième puce du noeud terminal.
    3. Onglet 3 : Render all cameras.

Note : Il s'agit de la même procédure que pour le deuxième passage

Les images sont alors enregistrées dans le dossier à la racine du projet nommé `TEMP`.

Note : Avec le développement des scripts envisagés, toutes ces étapes intermédiaires seront automatisées.
